# טיוטת מייל – אחרי דליפת מפתח Google/Firebase

**נושא:** טיפול בדליפת מפתח API – פרויקט cTeen Netivot (cteennetivot-216e7)

שלום,

קיבלנו התראת אבטחה מ-Google שמפתח ה-API של Firebase נחשף ברשת (הופיע בקוד ב-GitHub).

**מה כבר בוצע בפרויקט:**
- הוסר המפתח מהקוד. פרטי Firebase נטענים כעת ממשתני סביבה (קובץ `.env`) שלא מועלה ל-Git.
- נוסף קובץ `.env.example` עם שמות המשתנים – צריך להעתיק ל-`.env` ולהזין מפתח חדש.
- קבצי `.env` ו-`.env.local` נוספו ל-`.gitignore`.

**מה חובה לעשות עכשיו:**
1. **ב-Google Cloud Console:**  
   להיכנס ל-[Credentials](https://console.cloud.google.com/apis/credentials?project=cteennetivot-216e7), לפתוח את המפתח שנחשף ולבחור **"Regenerate key"** (או ליצור מפתח חדש ולמחוק את הישן).

2. **בפרויקט המקומי:**  
   ליצור קובץ `.env` בשורש הפרויקט (לפי `.env.example`) ולהזין את מפתח ה-API החדש ואת שאר ערכי Firebase.

3. **ב-GitHub:**  
   אם הריפו ציבורי – להסיר את המפתח מההיסטוריה (למשל עם `git filter-branch` או כלי כמו BFG), או לפחות לוודא שבקומיטים חדשים אין יותר מפתחות בקוד.

4. **הגבלות על המפתח ב-Console:**  
   להגביל את המפתח לפי יישום (Android/iOS/Web) ולפי API (Firebase רק) כדי לצמצם נזק במקרה של דליפה חוזרת.

אם צריך פרטים טכניים או עזרה בהגדרת `.env` / Regenerate – אפשר לחזור אליי.
